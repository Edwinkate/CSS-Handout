Styling die je toepast wordt toegepast in een bepaalde volgorde. Het kan voorkomen dat je styling toepast en vervolgens niet het gewenste resultaat ziet. Cascade en Inheritance zijn belangrijke concepten om te begrijpen hoe styling werkt. Lees hier in de officiële documentatie meer over cascade en inheritance. Zoek ook andere referenties om het concept te begrijpen. Leg in eigen woorden uit wat met cascade en inheritance wordt bedoeld. Maak voorbeelden om je antwoord uit te leggen.

Cascade behandelt de voorrang van stijlregels of CSS properties (de volgorde waarin eigenschappen worden toegepast in een html element) Het lost conflict situaties op.
Wanneer we een set regels definiëren voor een specifieke tag in een extern, intern en/of inline style, de browser maakt een: Virtual Declaration List door alle stijlregels samen te voegen. Dan volgt de Rules of Cascade die bepaalt welke regels uit de Virtual Declaration List worden toegepast op Specifieke tag.
De regels van Cascade zijn:
Latere properties overschrijven eerdere properties.
Meer specifieke selectoren overschrijven minder specifieke selectoren.

Inheritance behandelt hoe stijlen van een bovenliggend element naar de onderliggende elementen zijn overgebracht. Bepaalde properties zoals: font-family worden inherited. Als je een font family bepaalt in een body element dan zal dat inherited worden door alle elementen die in het body element zitten. Hetzelfde geld bijvoorbeeld ook voor de property color.
Niet alle properties worden inherited. Zoals bijvoorbeeld Background en height, die houden hun default van transparent en auto.
